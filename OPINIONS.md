# Chloe's opinions

A running list of Chloe's standing preferences for how work on this project
should go. These are opinions, not laws — but the default is to follow them, and
deviating is worth a sentence of justification.

## Engineering

- **When choosing between a shortcut and an elegant solution that takes more
  work, almost always avoid the shortcut.** The extra work is usually paid back:
  the elegant version is the one you don't have to unwind later, and the shortcut
  tends to defer a problem to a worse moment (often surfacing as a contradiction
  two steps downstream). If the shortcut is genuinely the right call, say why.
