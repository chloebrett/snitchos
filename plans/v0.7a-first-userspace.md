# v0.7a: first userspace process (built deliberately wrong)

**Status: SHIPPED (2026-06-10)** ‚Äî see the per-step DONE notes below. The
deliberate wrongness (ambient authority) was repaid by
[v0.7b-capabilities.md](v0.7b-capabilities.md), as designed.

The first program that runs in **U-mode**, with exactly **one syscall**,
under **ambient authority** ‚Äî no capability handle, no rights check.
Built the "Unix way" on purpose so v0.7b can feel the pain and refactor
it into capability invocation.

This is the privilege-boundary milestone, *not* the IPC or
multi-process milestone. One process. One syscall. Drop to U, trap back.

> Numbering: the roadmap split old "v0.7" into v0.7a (this) + v0.7b
> (capabilities); IPC moved to v0.8. The design docs
> (`docs/capability-system-design.md`, `docs/ipc-design.md`) were
> renumbered to match. Read the v0.7 conceptual overview before this.

## Goal

A `no_std` user program, embedded in the kernel image, runs in U-mode,
executes one `ecall` that asks the kernel to emit a telemetry metric on
its behalf, and the round trip is traced. By the end of the milestone
the process runs in its **own address space**.

The "wrongness" we are building on purpose: the telemetry syscall is
**ambient** ‚Äî any U-mode code can invoke it, the kernel performs no
authority check. v0.7b replaces this with a `TelemetrySink` capability
invocation.

## Non-goals (deferred, by milestone)

- **Capabilities / authority checks** ‚Äî v0.7b. The syscall is ambient.
- **IPC, endpoints, notifications** ‚Äî v0.8.
- **Multiple processes / scheduling user threads against each other** ‚Äî
  later. We have exactly one user process; the existing kernel tasks
  still run the show.
- **Preemption of userspace** ‚Äî v0.9. The user program runs
  cooperatively: it `ecall`s, we handle, we `sret` back. No timer
  preemption of U-mode (see "FP + preemption" gotcha).
- **FP register save/restore** ‚Äî v0.9 (lazy FP). The user program is
  integer-only for now; `sstatus.FS` stays Off.
- **Filesystem / loading from disk** ‚Äî v0.10. The program is
  `include_bytes!`'d into the kernel image.
- **ELF relocations / dynamic linking / PIE** ‚Äî out of scope
  permanently for this path. Static, position-dependent ELF linked at a
  fixed low-half VA.
- **`fork`/`exec`, signals, a POSIX surface** ‚Äî not a project goal.

## What we already own (reuse, don't rebuild)

- Trap entry/exit + `TrapFrame` + `decode_scause` (`kernel/src/trap.rs`,
  `trap.S`). `TrapCause::EnvCallFromUMode` (scause 8) already exists and
  is currently a `panic!` arm in `trap_handler` ‚Äî we grow it.
- `PtePerms` already has `U` (bit 4) and `X` ‚Äî the userspace bits exist.
- `kernel::mmu::map(va, pa, perms)` mutates `BOOT_PT_ROOT` ‚Äî Step A uses
  it directly. The pure walker `kernel_core::mmu::map(root_pa, ‚Ä¶, &mut
  mem)` is **root-parameterised** ‚Äî Step B builds on that.
- `tracing::emit_metric` / `register_counter` ‚Äî the telemetry syscall
  emits through these. **No `protocol::Frame` change** ‚Äî the metric
  frame already exists on the wire (contrast v0.5.x, which needed a new
  variant).
- The low half (Sv39 root PTE slots 0‚Äì255) is **entirely empty today**
  (kernel high-half at 256+, linear map at 322, heap at 256). User
  mappings drop in with no collision.

### Step 1 ‚Äî `trap.S`: kernel-stack switch via `sscratch`

Today `trap_entry` allocates the trap frame on the *current* `sp`
(`trap.S:52`). Safe only because every trap so far comes from S-mode,
where `sp` is already a kernel stack. A trap **from U-mode** arrives
with `sp` pointing at the *user* stack ‚Äî untrusted. We must switch to a
known kernel stack before touching memory.

**Convention** (software, not hardware): `sscratch == 0` while in the
kernel; `sscratch ==` this thread's kernel `sp` while in user.

Entry becomes:

```asm
trap_entry:
    csrrw sp, sscratch, sp      # swap. sp now = kernel sp (or old sp if sscratch was 0)
    bnez sp, .Lfrom_user        # sscratch was nonzero -> came from user
    # came from kernel: sscratch held 0, so sp now holds 0. Recover
    # the real sp by swapping back.
    csrrw sp, sscratch, sp      # sp = real kernel sp again; sscratch = 0 again
.Lfrom_user:
    # sp is now a good kernel stack in both cases.
    addi sp, sp, -288
    # ... existing GPR save, but DO NOT read the saved sp from sp; for
    #     the from-user case the user sp is in sscratch now ...
```

The from-user path must also save the **user `sp`** (currently sitting
in `sscratch` after the swap) into the frame's sp slot, and restore it
symmetrically on exit. The from-kernel path keeps today's "compute
caller sp = sp + 288" behaviour.

**Boot wiring:** set `sscratch = 0` once per hart at boot (kernel
convention). Each hart already has a kernel stack; this is a one-line
addition near trap-vector install. *Per-hart* ‚Äî `sscratch` is a CSR, so
naturally per-hart; set it on both harts.

**De-risking (revised ‚Äî no fake-U context):** with `sscratch = 0` in
the kernel, *every existing S-mode trap* (timer IRQ, IPI) now flows
through the rewritten entry, hits the `sscratch == 0` ‚Üí don't-switch
path, and must behave exactly as before. The **entire existing itest
suite is the regression gate** for this step. A fake-U context was
considered and rejected: it can drive the branch but tests the cheap
half (it can't exercise the real privilege drop or U-page fetch) and is
a footgun (nonzero `sscratch` in S-mode means a real IRQ mis-switches).
The U-path gets its first honest test in Step 4.

**Verification:** `cargo xtask itest --repeat 10` ‚Äî all existing
scenarios green. No new scenario here; this step is "didn't break the
kernel-trap path."

### Step 2 ‚Äî ELF parser in `kernel-core` (pure, host-tested)

ELF parsing is pure data transformation: bytes in, segments + entry
out. No `unsafe`, no MMIO. It lives in `kernel-core` and is host-tested
like everything else there. The `kernel`-side loader (Step 4) is the
thin part that maps + copies.

```rust
// kernel-core/src/elf.rs
pub struct LoadSegment {
    pub vaddr: usize,
    pub file_offset: usize,
    pub file_size: usize,   // bytes to copy from the image
    pub mem_size: usize,    // >= file_size; the tail is bss (zero-fill)
    pub flags: SegmentPerms, // R / W / X
}

pub struct LoadPlan {
    pub entry: usize,                 // e_entry
    pub segments: Vec<LoadSegment>,   // PT_LOAD only, in file order
}

pub fn parse(image: &[u8]) -> Result<LoadPlan, ElfError>;
```

Scope of the parser ‚Äî **only what loading needs**:
- ELF header: magic `\x7fELF`, class = ELF64, machine = RISC-V
  (`EM_RISCV = 243`), type = `ET_EXEC`, read `e_entry`,
  `e_phoff`, `e_phentsize`, `e_phnum`.
- Program header table: iterate, keep `p_type == PT_LOAD (1)`, read
  `p_offset / p_vaddr / p_filesz / p_memsz / p_flags`.
- No section headers, no symbols, no relocations, no dynamic table.

`ElfError` covers: bad magic, wrong class/machine/type, header offsets
out of bounds, `p_filesz > p_memsz`, segment file range out of bounds.
**This is a trust boundary even though the input is embedded** ‚Äî
validate, return `Result`, never panic. (When v0.10 loads from a file,
the same parser handles untrusted input unchanged.)

**Verification (host `cargo test -p kernel-core`):**
- Round-trip a hand-built minimal ELF byte buffer ‚Üí expected `LoadPlan`.
- bss case: `mem_size > file_size` parses with the right tail length.
- Rejection cases: bad magic, truncated PH table, `filesz > memsz`,
  out-of-bounds segment ‚Üí the right `ElfError`, no panic.
- Use a checked-in fixture: the actual compiled user program ELF (Step
  3) as `include_bytes!` in the test, asserting it parses.

### Step 3 ‚Äî the user program crate

A minimal `no_std` U-mode binary. Own crate (e.g. `user/hello`), own
linker script placing it at a fixed low-half VA, own `_start`, makes the
`ecall` directly. No std, no heap, integer-only.

```rust
#![no_std]
#![no_main]
// _start: load syscall number into a7, arg into a0, ecall, then loop.
// For v0.7a the program does: ecall(SYS_EMIT_METRIC, some_value); loop {}
```

Decisions:
- **Link VA:** fixed, e.g. `0x1000_0000` (well clear of everything;
  must match between Step A and Step B ‚Äî see the no-rework argument in
  the overview). Linker script sets the load address; the program is
  position-dependent, so no relocations.
- **Stack:** a `.bss` array in the program, `sp` set to its top in
  `_start`. (Or the kernel maps a separate user-stack page and sets the
  initial user `sp` in the trap frame ‚Äî pick during impl; a `.bss`
  stack is simplest and falls out of the ELF loader for free.)
- **The `ecall` ABI:** `a7` = syscall number, `a0..a5` = args, return
  value in `a0` (Linux/SBI convention; cheap and familiar).

**xtask:** add a build step that compiles `user/hello` for its bare
U-mode target *before* the kernel, and the kernel `include_bytes!`s the
resulting ELF. Mirror how the itest-workloads build is sequenced.

**Verification:** the artifact exists and `kernel-core::elf::parse`
accepts it (the Step 2 fixture test). No runtime behaviour yet.

### Step 4 ‚Äî STEP A: privilege drop in the shared address space (first light)

> **Already in place from Step 3** (don't rebuild): the embedded
> `user::HELLO_ELF` bytes, the page-separated segment layout, and the
> `kernel_core::elf` parser. Step 4 is now purely: parse ‚Üí map ‚Üí enter ‚Üí
> handle the syscall ‚Üí return.

The novel, scary part ‚Äî isolated. Map the program into the **boot page
table's low half**, drop to U-mode, handle the `ecall`, return.

**Load (kernel side, thin):** for each `LoadSegment` in the `LoadPlan`:
1. `frame::alloc_zeroed` enough frames for `[vaddr, vaddr + mem_size)`.
2. `kernel::mmu::map(vaddr_page, pa, U | seg_perms)` per page ‚Äî `U`
   always set; R/W/X from the segment (text `R|X`, rodata `R`, data
   `R|W`). Real W^X per segment.
3. Copy `file_size` bytes from the embedded image into the mapped pages
   (via the linear-map VA of the frame, `pa_to_kernel_va`), leave the
   bss tail zero (frames came zeroed).

**Enter U-mode:** build a `TrapFrame` (or reuse the trap-return path)
with:
- `sepc = plan.entry`
- `sstatus.SPP = 0` (return to U), `SPIE = 1` (enable interrupts after
  sret), `SUM` left **0** (kernel must not be reading user pages yet),
  `FS = Off`.
- `sscratch =` the kernel stack to use for traps from this user thread.
- `sp` (in the frame / the user's a-priori sp) = top of the user stack.
- `a7 / a0` set by the program itself; kernel just needs entry + sp.

then `sret`. The CPU drops to U-mode at `plan.entry`.

**Handle the syscall** ‚Äî grow the existing dispatcher:

```rust
// trap.rs trap_handler, currently panics on EnvCallFromUMode
TrapCause::EnvCallFromUMode => handle_user_ecall(frame),
```

```rust
fn handle_user_ecall(frame: &mut TrapFrame) {
    match frame.a7 {
        SYS_EMIT_METRIC => {
            // AMBIENT: no capability check. This is the wrongness.
            let value = frame.a0 as i64;
            tracing::emit_metric(USER_METRIC_ID, value);
            frame.a0 = 0; // success
        }
        _ => frame.a0 = u64::MAX, // unknown syscall
    }
    frame.sepc += 4; // CRITICAL: step past the ecall, else we re-trap it forever
}
```

`USER_METRIC_ID` is a kernel-registered counter, e.g.
`snitchos.user.telemetry_total`, registered once at boot. The metric
frame already exists on the wire ‚Üí **no protocol change**.

**Gotchas baked into this step:**
- **`sepc += 4`** after an `ecall`: `sepc` points *at* the `ecall`;
  without advancing, `sret` re-executes it ‚Üí infinite syscall loop.
  (Compressed `c.ecall` doesn't exist; `ecall` is always 4 bytes.)
- **The instruction fetch needs `X` *and* `U`** on the entry page. Miss
  `U` ‚Üí instruction-page-fault the instant we land in U. Miss `X` ‚Üí
  same. Both are easy to drop.
- **`SUM` (Supervisor User Memory access):** with `SUM=0`, the kernel
  faults if it dereferences a user (`U`) page. For v0.7a the kernel does
  not need to read user memory (the metric value comes in a register),
  so keep `SUM=0` ‚Äî it catches accidental user-pointer derefs. If a
  later syscall must read a user buffer, set `SUM` only around that
  access. Flag for v0.7b/v0.8.
- **`sstatus.SPIE`** controls whether interrupts are on *after* `sret`.
  Set it so the timer keeps firing while user runs ‚Äî otherwise the
  heartbeat stalls whenever userspace is on-CPU.
- **Don't preempt U-mode yet.** A timer IRQ *will* fire while the user
  program loops; the existing handler sets `TICK_PENDING` and returns ‚Äî
  it does **not** context-switch. Fine: we return to the same user
  thread. (Preempting U requires the full-trap-frame switch ‚Äî v0.9.)
  But confirm the IRQ-from-U path also goes through the Step 1
  stack-switch correctly (it does: same `sscratch` convention).

**Where the user program runs in the boot flow:** simplest is a kernel
task (or `kmain` directly) that does the load + `sret` once, after MMU
and tracing are up. The user program ecalls once and `loop {}`s; the
ecall returns to it; control stays in U until the next trap. Since the
kernel never re-enters this user thread voluntarily after the first
sret in v0.7a, the integration is: enter once, observe the metric, the
timer IRQ keeps the rest of the kernel alive.

**Verification ‚Äî new scenario `userspace-emits-telemetry`:**
- Assert a `snitchos.user.telemetry_total` metric frame appears within
  N seconds (proves: privilege drop happened, ecall trapped back,
  syscall handled, sret returned).
- Assert the kernel keeps heartbeating afterward (proves: we didn't
  wedge the kernel / the IRQ-from-U path works).
- `--repeat 10`.

This is **first light** ‚Äî the milestone's headline trace.

### Step 5 ‚Äî STEP B: per-process address space

Give the process its own root page table. The *only* changes from Step
A (per the no-rework argument in the overview, holding the user VA
fixed):

1. **Build a fresh root.** `frame::alloc_zeroed` one frame for the new
   root page table.
2. **Share the kernel high-half in.** Copy the kernel's top-level root
   entries (the high-half slots: kernel image, linear map, heap ‚Äî
   indices ‚â• 256) from `BOOT_PT_ROOT` into the new root. This is the
   Q27a decision made concrete: kernel mapped into every address space,
   so traps need no page-table switch. (These high-half leaves are
   shared, not copied deep ‚Äî copying the *root* entries suffices since
   the sub-tables are shared.)
3. **Map the user segments into the new root's low half**, not the boot
   root. Same `LoadPlan`, same VAs, same perms ‚Äî different root. Needs a
   kernel-side `map`-into-arbitrary-root wrapper: today `kernel::mmu::map`
   is hardcoded to `BOOT_PT_ROOT`; the pure `kernel_core::mmu::map` is
   already root-parameterised, so the wrapper is thin (point it at the
   new root's PA). **This wrapper is the one genuinely new primitive in
   Step B.**
4. **Switch `satp` on entry**, just before `sret`: write `satp` with
   Sv39 mode + the new root's PPN, then `sfence.vma`. Two instructions
   added to the entry path. (Mirror the `csrw satp` / `sfence.vma`
   sequence in `mmu::enable`.)

The trap path is **unchanged** ‚Äî kernel is mapped in the new space too,
`sscratch` works identically. The hard parts (Step 1, Step 4) are
untouched.

**Address-space struct:** a minimal `AddressSpace { root_pa: usize }`
(grows into the `AddressSpace` capability object in v0.7b). Associate it
with the user thread/process. A "process" in v0.7a = one thread + one
`AddressSpace`; the full process object is a v0.7b concern.

**Isolation check ‚Äî this is what Step B *buys*:** add a second mapping
proving isolation is real, e.g. assert the user program **cannot** read
a kernel high-half address (a load from a kernel VA faults to S-mode
with a load-page-fault from U). This is also the seed of the v0.7b
story: today only the page table stops you; capabilities will add the
authority layer on top.

**Verification:**
- Re-run `userspace-emits-telemetry` under per-process `satp` ‚Äî still
  green (proves the relocation + satp switch didn't regress first
  light).
- New scenario `userspace-cannot-touch-kernel`: the user program
  attempts a load from a kernel VA; assert the kernel observes a
  U-mode load-page-fault (a new traced event or a counter) rather than
  the load succeeding. Proves isolation.
- `--repeat 10` on both.

## Wire format changes

**None for Step 4.** The telemetry syscall emits an existing metric
frame. Step 5's isolation check *may* want a new event/counter for
"U-mode page fault observed" ‚Äî if so, add it as a new `protocol::Frame`
variant **after** existing variants (postcard is positional) and update

(`snitchos.user.faults_total`) over a new variant if it suffices ‚Äî no
protocol commit.

## Key gotchas (collected)

1. **`sepc += 4`** after `ecall` or the syscall re-traps forever.
2. **Entry page needs `U` *and* `X`**; data pages `U` (+`R`/`W`).
   Missing `U` faults the instant U-mode touches the page.
3. **`sscratch = 0` in kernel** is load-bearing for the Step 1 branch;
   set it at boot on every hart, and a real IRQ while it's nonzero in
   S-mode mis-switches.
4. **`SPP=0`, `SPIE=1`, `SUM=0`, `FS=Off`** in `sstatus` before `sret`.
5. **`sfence.vma` after the `satp` switch** (Step 5) ‚Äî stale TLB
   otherwise. The boot enable sequence is the template.
6. **No formatted `println!` before the higher-half mapping** ‚Äî not new,
   but the user-load code runs late so it's fine; noted for safety.
7. **Don't try to preempt U-mode** ‚Äî the cooperative `switch` can't
   carry a preempted user thread (caller-saved regs + FP); that's v0.9.

## HAL note

Trap entry, the `sscratch` dance, the `satp` write, and `sret` are all
RISC-V-specific and would sit below the HAL line
(`docs/parked-memory-model-and-hal.md`). The ELF parser and the
load-plan-to-map loop are portable (above the line). We have no
`arch-riscv64` crate yet; keep the asm-touching code in `kernel/` and
the pure parser in `kernel-core/`, consistent with the current split.
The HAL crate extraction is parked, not part of v0.7a.

## Open questions to settle during impl

- **User stack:** program-internal `.bss` stack (simplest, free from the
  ELF load) vs. a kernel-mapped dedicated user-stack page (cleaner
  separation, lets us guard-page it later). Lean `.bss` for v0.7a.
- **Who runs the load+enter:** a dedicated kernel task vs. inline in
  `kmain` after tracing is up. Lean a small task so it's a clean unit.
- **Syscall number space:** single `SYS_EMIT_METRIC = 0`, or reserve a
  tiny enum now? Lean a tiny `enum Syscall` in a shared no_std module so
  the user crate and kernel agree without magic numbers.
- **Metric semantics:** counter (`telemetry_total`, value ignored / +1)
  vs. gauge (report the `a0` value). Lean counter for v0.7a ‚Äî proves the
  path; a value-carrying syscall is a natural v0.7b refinement.
- **`AddressSpace` home:** `kernel/` (touches `satp`) vs. a portable
  descriptor in `kernel-core` with the `satp` write in `kernel/`. Lean
  the latter to keep the split clean for the eventual HAL.
- **Isolation-fault signalling:** new `Frame` variant vs.
  `faults_total` counter. Lean counter (no protocol commit) unless the
  trace story wants a first-class event.

## What this does NOT unlock

- A second user process. One address space, one thread. Multi-process
  arrives with IPC (v0.8) needing somewhere to talk.
- Userspace calling back into itself or being scheduled against kernel
  tasks fairly. The user thread runs until it traps; we don't
  round-robin it against task_a/task_b in v0.7a.
- Any authority model. *Anyone* in U-mode can call the telemetry
  syscall ‚Äî that's the wrongness v0.7b exists to fix.
- Surviving a preempting timer interrupt mid-computation as a saved/
  restored user context. We return to the same user PC; we don't
  switch away. v0.9.
