## The object in one paragraph
A `Notification` is one machine word of pending-signal bits plus at most one
parked waiter. `Signal` OR-s a mask into `pending` and wakes any waiter (never
blocks). `WaitNotify` returns-and-clears `pending` if nonzero, else parks the
caller and blocks. Coalescing falls out (N signals before a wait collapse to one
wake of OR-ed bits) → no queue, no per-signal kernel memory. Cap-gated by
disjoint `SIGNAL`/`WAIT` rights. Every signal/wake is a wire frame.

## TDD discipline (per project rules)
Each increment is **RED first, in its own edit** (failing test), then the
minimum GREEN, then MUTATE (`cargo xtask mutants` on the touched core) and kill
survivors, then assess refactor. Never batch the test and impl into one edit,
even when the pattern feels obvious. Host-testable logic lands in `kernel-core`
and is covered by `cargo test -p kernel-core`; kernel-side asm/CSR/MMIO wiring is
covered transitively by the QEMU itest.

## Increment 1 — `kernel_core::notify` pure core (host-tested)
Mirror `kernel_core::reap` exactly: pure data + bookkeeping, no `unsafe`, no MMIO.

New file `kernel-core/src/notify.rs`, exported from `lib.rs`.

```rust
pub enum SignalStep { Woke(TaskId), NoWaiter }
pub enum WaitStep   { Ready(u64), Block }

pub struct Notification { pending: u64, waiter: Option<TaskId> }

impl Notification {
    pub const fn new() -> Self;
    pub fn signal(&mut self, mask: u64) -> SignalStep; // pending |= mask; Woke(w) if a waiter is parked, else NoWaiter
    pub fn wait(&mut self, caller: TaskId) -> WaitStep; // pending!=0 → Ready(take pending, clear); else park caller, Block
}
```

RED→GREEN test list (one test per behaviour, each written before its code path):
1. `wait_with_no_pending_blocks` → `Block`, caller recorded as waiter.
2. `signal_then_wait_returns_the_bits` → `signal(0b1)` then `wait` → `Ready(0b1)`.
3. `wait_clears_pending` → after a `Ready` wait, a second `wait` → `Block` (read-and-clear, not re-report).
4. `signals_coalesce` → `signal(0b001)`, `signal(0b100)` before any wait → one `wait` → `Ready(0b101)`.
5. `signal_with_a_parked_waiter_wakes_it` → `wait` (Block), then `signal(0b1)` → `SignalStep::Woke(that_caller)`; the bits are pending for the woken task's re-`wait`.
6. `signal_with_no_waiter_is_noop_wake` → `signal` with nobody parked → `NoWaiter`, bits still pending.
7. `second_waiter_is_rejected` → one waiter parked, a *different* task `wait`s → distinct outcome (NOT silent overwrite). **Decision to encode here:** return a `WaitStep::Busy` (or `Refused`) variant rather than overwriting `waiter` — this is the lesson from `ReapTable.waiters`' single-slot overwrite (v0.12 edge #3). The kernel maps `Busy` → `SyscallRefused`.

MUTATE: run mutants on `notify.rs`; the OR-accumulate, the take-and-clear, and the waiter-occupied check are the load-bearing lines — confirm each has a killing test.

## Increment 3 — the live table + kernel block/wake wiring
Kernel side owns the live notifications behind a `Mutex`, exactly as `REAP` owns `ReapTable`.

- A `NOTIFY` static: `Mutex<NotifyTable>` mapping `NotificationId → Notification` (+ an id allocator). `NotifyTable` itself is host-testable in `kernel-core` (the map + allocate; reuse the `BTreeMap` pattern from `reap`).
- `kernel::sched` (or a small `kernel::notify` module): `signal(id, mask)` → on `Woke(w)` call `sched::wake(w)`; `wait(id, caller)` → on `Block` call `block_current()`, on `Ready(bits)` return the bits. Reuses `block_current`/`wake` **verbatim** — no new park/unpark machinery.

Covered transitively by the itest in increment 6 (no host test for the asm-adjacent wiring; the pure `NotifyTable` map logic gets its own kernel-core tests).

## Increment 4 — syscalls: `NotifyCreate` / `Signal` / `WaitNotify`
Append to `abi::Syscall` past `Wait = 18` (numbers 19/20/21 — appending is free; syscall numbers are not frozen, only the postcard frame format is). Add `from_usize` arms (host-tested in `abi`).

Dispatch in `kernel/src/syscall/` (new `notify.rs`, wired into the trap dispatch):
- `NotifyCreate` (19): mint a fresh `SIGNAL|WAIT` `Notification` cap into the caller's table; return the handle in `a0`. Ambient (making your own notification needs no prior cap), like `MapAnon`.
- `Signal` (20): resolve `a0` cap, require `SIGNAL`, OR `a1` mask, wake any waiter, `a0 = 0`. Refuse (snitch `SyscallRefused`) on bad handle / missing right.
- `WaitNotify` (21): resolve `a0` cap, require `WAIT`; `Ready(bits)` → `a0 = bits`; `Block` → `block_current()` (return value set on wake); `Busy` (second waiter) → refuse + snitch.

Every refusal path emits `SyscallRefused` + bumps the refusal counter — the project invariant, never a silent failure.

## Increment 5 — wire frames + runtime bindings
- `protocol`: add `Frame::NotifySignal { notification, mask, from_task }` and `Frame::NotifyWait { notification, bits, to_task }` (**appended** — never reorder existing variants; postcard is positional). Roundtrip tests in `protocol` (the established `Frame` test pattern). Update `OwnedFrame::from_borrowed` arms.
  - Frame-budget note from the design doc: emit `NotifySignal` on the empty→nonempty transition + on each wake, not on every redundant OR — keeps a future high-rate IRQ source from flooding the wire. Encode that in the kernel emit site.
- `user/runtime/src/lib.rs`: `notify_create() -> Handle`, `signal(h, mask)`, `wait_notify(h) -> u64` — siblings of the existing `wait(child) -> i32` / `exit_with` bindings.

## Increment 6 — itest scenario (the proof on the wire)
A fresh minimal consumer — *not* child-exit (that path already works and stays).

New `workload=notify-smoke` (kernel-core `WorkloadKind` variant + parse arm, host-tested; dispatch in `kmain`/spawn layout per the runtime-workload pattern):
- Task `A` holds a `SIGNAL` cap, task `B` holds the `WAIT` cap on the same notification (parent creates + delegates the two ends).
- `B` `wait_notify`s (parks); `A` `signal`s a known mask; `B` wakes with exactly that mask.

Scenario `notify-signal-wakes-waiter` in `xtask/src/itest/scenarios.rs` (registered in `itest.rs::SCENARIOS`): assert `B` observed the right bits (via a marker metric or `Log`) **and** a `NotifySignal`→`NotifyWait` pair appears on the wire with matching `notification` id and `to_task == B`. This is the post angle made testable: the async edge a synchronous trace can't show.

Gate before calling it done: `cargo xtask itest --repeat 10` clean (single-run-green hid the v0.6 flakes).

## Out of scope (documented growth points, not silent gaps)
- **Folding child-exit onto the notification (Option B)** — after the device-IRQ consumer validates the generic shape.
- **Multi-waiter fan-out** (one signal wakes N) — needs a waiter list, not a slot; increment 1's `Busy` rejection is the placeholder invariant.
- **Badged notifications** (distinguish *who* signalled) — the signal mask covers single-process demux without it.
- **Non-blocking `poll`** flag on `WaitNotify` — additive when an event loop over several sources needs it.
- **IRQ→notification binding** — the kernel trap handler signalling a notification a driver waits on. The *real* point of the primitive; lands with device drivers (post-v1.0 HAL/arcade arc), reusing this object unchanged.
