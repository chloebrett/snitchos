The cooperative scheduler becomes *time-sliced*: a userspace task that
never calls `yield_now` and never syscalls can no longer hog the CPU —
the timer takes it back. The headline: **"making the scheduler take the
CPU back."**

## Scope decision: preempt userspace, keep the kernel non-preemptible

Two models were on the table (see the §"Why these choices" recap):

- **Non-preemptible kernel** — the timer reschedules *only* when it
  interrupted **user** code. Kernel code is never involuntarily
  suspended; it keeps the cooperative "I have exclusive access until I
  `yield_now`" guarantee. Predicate: `SPP == User`.
- **Preemptible kernel** — the timer reschedules anywhere preemption
  isn't explicitly disabled. Predicate: `preempt_count == 0`. Requires a
  safety-critical audit of every lockless kernel critical section.

**v0.8 takes the non-preemptible-kernel model.** It delivers the entire
headline once the demo workers are userspace, keeps the kernel's
internal cooperative invariant intact (so the re-entrant-reschedule
corruption class mostly evaporates), and dodges the low-probability
re-entrancy flake class. The upgrade to a preemptible kernel is a
**one-predicate flip** (`SPP == User` → `preempt_count == 0`) plus the
lockless-section audit, deferred to its own future milestone — *not* an
omission. All the hard machinery built here (full-trap-frame switch,
timer→reschedule wiring, slice accounting) carries over unchanged.

## Prerequisite (upstream, not owned by this plan)

Preemption is only observable on a task that **won't cooperate**. Today's
demo workers (`task_a`, `task_b`) are kernel-mode threads — a
non-preemptible kernel can't preempt them, and they yield voluntarily
anyway. So v0.8 depends on:

- **`task_a` / `task_b` refactored into userspace processes.** They were
  kernel-mode as a pre-userspace stopgap (v0.5, before v0.7 userspace
  existed). This is a v0.7 follow-on and likely its own plan
  (`plans/userspace-demo-workers.md`).
- **At least one non-yielding, CPU-bound userspace itest workload** — a
  tight `loop {}` in U-mode that never syscalls — as the preemption test
  fixture. Selected via `workload=user-hog` (runtime workload registry).

If that refactor isn't done first, Step 3 below introduces the *minimal*
version (one CPU-hog user task) so v0.8 can be tested; the broader
worker migration can land separately.

## Goal

A userspace task that never yields is descheduled by the timer after its
slice expires, another runnable task makes progress, and a
`ContextSwitch { reason: Preempt }` frame proves it on the wire.

## Decisions locked in

| decision | choice |
|---|---|
| What is preemptible | **Userspace only.** Kernel stays cooperative (non-preemptible). |
| Reschedule predicate | `quantum_expired && SPP == User` |
| Switch mechanism | **Layered (xv6-style).** Reschedule rides the existing cooperative `switch` (sched.S), called from inside the timer handler. The preempted user task's full state lives in its `TrapFrame` on its kernel stack; `switch` swaps `sp` and carries it along. **sched.S is not modified.** |
| Slice accounting | Reuse `CURRENT_TASK_ENTRY_TICK` (already tracks when the running task became Running). `quantum_expired = now - entry >= QUANTUM_TICKS`. |
| Quantum | Fixed `QUANTUM_TICKS` (in heartbeat-timer ticks) for v0.8. Per-priority quanta deferred to the priorities follow-on. |
| Decision logic location | Pure function in `kernel-core::sched` (host-tested), mirroring `heap::watermark_grow_decision`. |
| Wire format | Append `Preempt` to the `ContextSwitch` reason enum (postcard is positional — **append only**, never reorder). |
| Telemetry | `snitchos.sched.preemptions_total`; slice-length histogram is a stretch. Emitted via the **deferred-emission** pattern (bump atomic in the handler, drain in heartbeat) — never emit a frame from inside the timer handler. |
| `preempt_count` hooks | **Deferred.** With the `SPP == User` gate, the timer only reschedules at the user→kernel trap boundary (a safe point, no locks held), so a preempt-disable counter isn't needed for v0.8 correctness. It lands with the kernel-preemptible flip. |
| Priorities | Deferred to a follow-on PR / sub-milestone (`v0.8b`). v0.8 ships round-robin time-slicing. |

## Why these choices (conceptual recap)

- **Two register-saving disciplines already exist, and we compose them
  by layering.** The cooperative `switch` saves 14 callee-saved regs
  because it's a *function call* (the ABI already spilled live
  caller-saved values). `trap_entry` saves the *full* GPR set + `sepc` +
  `sstatus` because an interrupt lands at an arbitrary PC with live
  caller-saved regs. Preemption is **not** a bigger switch — the timer
  trap already builds the full `TrapFrame`; we just make the handler
  resume a *different* task by calling the existing cooperative `switch`.
  The full frame parks on the task's own kernel stack, invisible to the
  scheduler, which still only ever manipulates `TaskContext`s.
- **Why per-task kernel stacks + `sscratch` make this nearly free.** Each
  task already owns a 16 KiB kernel stack, and the `sscratch`
  kernel-stack-top convention (trap.rs) already does the user→kernel
  stack swap. The preempted frame lands on the task's stack; `switch`
  swapping `sp` carries it; switching back unwinds it and `sret`s to the
  exact user PC.
- **Why non-preemptible kernel now.** The runaway tasks are userspace,
  so `SPP == User` is sufficient to take the CPU back from them. Kernel
  threads (`idle`, `main`) yield by construction, so they never need
  forcing. This keeps the cooperative exclusivity guarantee inside the
  kernel and avoids the preempt-disable audit.
- **`yield_now` stays.** It's the *voluntary relinquish* primitive
  (cheap 14-reg path at a known-safe point); preemption is the
  *involuntary preempt* primitive. v0.9 IPC `block`/`wake` are
  `yield_now`'s descendants — blocking on an empty channel must give up
  the CPU *immediately*, which preemption alone can't express.

### Step 1: Slice-expiry decision as pure host-tested logic

**Acceptance criteria**: A pure function in `kernel-core::sched` returns
`true` when `now - entry >= QUANTUM_TICKS`, `false` below it, and handles
the boundary (`== QUANTUM_TICKS` → true) and `now < entry` (clock
non-monotonic guard → false, never panic). No kernel behaviour change
yet. *Present criteria and confirm before coding.*
**RED**: `kernel-core` unit tests for the four cases above.
**GREEN**: the comparison, mirroring `heap::watermark_grow_decision`.
**MUTATE**: run `mutation-testing` — verify the boundary `>=` and the
monotonic guard both have killing tests.
**KILL MUTANTS**: add cases for any survivor (esp. `>` vs `>=`).
**REFACTOR**: only if it adds value.
**Done when**: host tests + mutation report green, human approves commit.

> **PR boundary** — pure logic, independently mergeable.

### Step 4: Timer preempts a runaway userspace task (the milestone heart)

**Acceptance criteria**: Under `workload=user-hog` + a runnable peer, the
peer makes progress and a `ContextSwitch { reason: Preempt }` frame
appears within N seconds; the previously-failing Step 3 scenario now
passes (peer progresses). Kernel code is still never preempted (a timer
landing in S-mode does not reschedule). *Confirm before coding.*
**RED**: itest scenario `preempt-runaway-user-task` — boot `user-hog` +
peer, assert (a) a `ContextSwitch{Preempt}` on the wire, (b) peer
progress, (c) both endpoints are known task ids.
**GREEN**: thread the `TrapFrame` into the timer arm (`handle_timer`
gains the frame, like `handle_user_ecall`); when
`quantum_expired(CURRENT_TASK_ENTRY_TICK, now) && (sstatus & SSTATUS_SPP)
== 0`, invoke the scheduler's reschedule path (the existing cooperative
`switch`, layered). Stage the `Preempt` ContextSwitch via the
deferred-emission atomic, not from inside the handler.
**MUTATE**: the decision call site + the `SPP` gate (mutate the mask /
the `==0` to confirm a test catches "preempts kernel code too").
**KILL MUTANTS**: add a scenario asserting a kernel-mode timer does NOT
reschedule if any survivor shows the gate is untested.
**REFACTOR**: assess; only if valuable.
**Done when**: scenario green under `cargo xtask itest --repeat 10`
(per the commit-gate rule — single-run-green hid prior flakes), mutation
report reviewed, human approves commit.

> **PR boundary** — preemption works end-to-end. This is a shippable
